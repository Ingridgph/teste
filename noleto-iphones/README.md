O **Noleto iPhones** é uma **landing page de e-commerce** moderna e imersiva desenvolvida com as tecnologias mais atuais do ecossistema React. O projeto combina **animações 3D interativas**, **smooth scrolling**, **sistema de temas light/dark** e uma **experiência de compra direta via WhatsApp**, entregando uma vitrine digital premium para produtos da Apple no mercado brasileiro.

A proposta central é unir estética de alto padrão — inspirada no design da própria Apple — com performance técnica de ponta, utilizando **Next.js 16**, **Three.js** para renderização 3D, **GSAP** para animações cinematográficas e **Framer Motion** para micro-interações polidas.

### 🎪 Destaques do Projeto

| Característica | Detalhe |
|---|---|
| 🏗️ **Arquitetura** | Next.js App Router com Server/Client Components |
| 🎯 **Foco** | Landing page conversível + catálogo de produtos |
| 📱 **Responsivo** | Mobile-first, grid adaptativo 1/2/3 colunas |
| 🌓 **Tema** | Light/Dark com detecção automática de preferência |
| 🛒 **Checkout** | Pedido formatado enviado diretamente ao WhatsApp |
| 🔒 **Validação** | React Hook Form + Zod com feedback em tempo real |
| ⚡ **Animações** | GSAP ScrollTrigger + Framer Motion + CSS 3D |
| 🌐 **3D** | Three.js com @react-three/fiber e HDR environment |

### 🎨 Interface & Experiência Visual

- **🍎 iPhone 3D Interativo (CSS Pure)** — Modelo do iPhone renderizado com perspective 3D, `Dynamic Island`, botões laterais e efeito de glow animado na tela. Responde ao movimento do mouse via GSAP (`rotateX`/`rotateY` em tempo real) e cria parallax no scroll.
- **🌐 Three.js Canvas** — Hero alternativo com modelo 3D usando `RoundedBox`, `metalness/roughness` para realismo de material, animação `Float`, `Environment HDR` e rotação/escala controladas pelo scroll.
- **🎬 Animações GSAP** — Entrance animations, stagger sequencial, ScrollTrigger com `pin`, `scrub`, easings `elastic`, `back` e `power3`, timelines sincronizadas com o scroll.
- **💨 Smooth Scroll (Lenis)** — Scrolling suavizado com easing customizado, integrado ao GSAP ScrollTrigger via ticker para consistência perfeita entre as bibliotecas.
- **🪟 Navbar Glassmorphism** — Barra fixa com `backdrop-filter: blur(20px)`, `color-mix` para transparência adaptativa ao tema, ícones sociais, toggle de tema e links de navegação com hover states.
- **✨ Framer Motion** — Micro-interações e transições entre páginas com `AnimatePresence`.

### 🛒 E-commerce & Compras

- **📦 Carrossel de Produtos** — Auto-rotação a cada 4 segundos com pausa inteligente no hover, navegação por setas e indicadores dot. Animação de entrada GSAP com `opacity`, `x` e `scale` em stagger.
- **🛍️ Carrinho Persistente** — Adição de produtos com animação `bounce` no ícone, contador de itens com badge animado no botão flutuante do WhatsApp.
- **📱 Checkout via WhatsApp** — Geração automática de mensagem de pedido formatada (lista de produtos, quantidades, preços e total) enviada diretamente ao WhatsApp do vendedor.
- **🔎 Busca e Filtros** — Campo de busca com debounce filtrando o catálogo em tempo real por nome do produto, combinado com filtros de categoria.

### 📝 Formulários & Contato

- **✅ Validação em Tempo Real** — React Hook Form + Zod schema com `mode: 'onChange'`, exibindo erros inline abaixo de cada campo.
- **🎭 Contact Palette** — Modal de contato estilo command-palette (ativado com `Ctrl+K` / fechado com `Esc`), com `backdrop-filter: blur`, animação de entrada via GSAP e formulário completo.
- **📄 Página de Contato** — Rota `/contato` com bio do fundador, história da loja, informações de atendimento e formulário que compõe mensagem para WhatsApp.

### 🌓 Sistema de Temas

- **Detecção Automática** — Respeita `prefers-color-scheme` do sistema operacional na primeira visita.
- **Persistência** — Preferência salva no `localStorage` do navegador.
- **Anti-Flash (FOUC Prevention)** — Script inline executado no `<head>` antes da renderização para aplicar o tema correto sem piscar.
- **Transição Suave** — `transition: background 0.5s, color 0.5s` em todas as variáveis CSS.
- **Toggle na Navbar** — Botão acessível com ícone ☀️/🌙 para alternar entre os temas.

## 🚀 Como Executar

### Pré-requisitos

```bash
# Versões recomendadas
node --version  # >= 18.17.0 (LTS recomendado: 20.x)
npm --version   # >= 9.0.0
```

> ⚠️ **Atenção:** Este projeto usa Next.js 16 e React 19. Certifique-se de usar Node.js 18+ para compatibilidade total.

### Instalação

**Passo 1 — Clone o repositório**
```bash
git clone https://github.com/seu-usuario/noleto-iphones.git
cd noleto-iphones
```

**Passo 2 — Instale as dependências**
```bash
npm install
# ou com yarn:
yarn install
# ou com pnpm:
pnpm install
```

**Passo 3 — Configure as variáveis de ambiente (opcional)**
```bash
# Crie o arquivo .env.local na raiz do projeto
cp .env.example .env.local

# Edite com suas configurações:
# NEXT_PUBLIC_WHATSAPP_NUMBER=5511999999999
# NEXT_PUBLIC_SITE_URL=https://seu-dominio.com.br
```

**Passo 4 — Execute em desenvolvimento**
```bash
npm run dev
```

**Passo 5 — Acesse no navegador**
```
http://localhost:3000        → Home (Hero 3D + Catálogo)
http://localhost:3000/contato → Página de contato
```

```bash
# 🔥 Servidor de desenvolvimento com hot reload
npm run dev

npm run build
```

## 🤝 Como Contribuir

Contribuições são **muito bem-vindas**! Seja uma correção de bug, uma nova funcionalidade ou melhoria na documentação.

### Fluxo de Contribuição

```bash
# 1. Fork o repositório
# (botão Fork no GitHub)

# 2. Clone seu fork
git clone https://github.com/SEU-USUARIO/noleto-iphones.git
cd noleto-iphones

# 3. Crie uma branch para sua feature
git checkout -b feat/minha-nova-feature
# ou para bugs:
git checkout -b fix/correcao-do-bug

# 4. Instale as dependências
npm install

# 5. Faça suas alterações e teste localmente
npm run dev

# 6. Verifique o linting
npm run lint

# 7. Commit seguindo o padrão Conventional Commits
git add .
git commit -m "feat: adiciona galeria de fotos na página de produto"

# 8. Push para sua branch
git push origin feat/minha-nova-feature

# 9. Abra um Pull Request no GitHub
```

### Padrão de Commits (Conventional Commits)

| Prefixo | Quando usar |
|---|---|
| `feat:` | Nova funcionalidade |
| `fix:` | Correção de bug |
| `style:` | Mudanças de CSS/estilo sem lógica |
| `refactor:` | Refatoração sem novo comportamento |
| `docs:` | Atualização de documentação |
| `test:` | Adição ou correção de testes |
| `chore:` | Configuração, deps, tooling |
| `perf:` | Melhoria de performance |

### Diretrizes de Código

- ✅ **TypeScript estrito** — Sem `any` implícito, tipagem completa
- ✅ **Componentes funcionais** — Hooks ao invés de classes
- ✅ **'use client' seletivo** — Server Components quando possível
- ✅ **Cleanup de efeitos** — `useEffect` com return cleanup para GSAP/Lenis
- ✅ **Responsivo** — Mobile-first com Tailwind
- ✅ **Acessível** — ARIA labels, foco visível, semântica HTML
